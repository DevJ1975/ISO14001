## Aspect vs impact

- An **aspect** is an element of the organization's activities, products or services that can interact with the environment (for example, *discharge of wastewater*).
- An **impact** is the resulting change to the environment (for example, *degradation of water quality*).

## Determining significance

Assess each aspect using consistent criteria, typically combining:

- **Severity / scale** of the potential impact;
- **Likelihood / frequency** of occurrence;
- **Legal and other obligations** that apply;
- **Interested-party concerns**;
- conditions: **normal, abnormal and emergency**.

Aspects that score above your threshold become **significant aspects** and must be controlled and considered when setting objectives.

## What to record per aspect

| Field | Notes |
| --- | --- |
| Activity / product / service | The source of the aspect. |
| Aspect | The interaction with the environment. |
| Impact | The environmental consequence. |
| Lifecycle stage | Acquisition, design, operation, end-of-life. |
| Condition | Normal / abnormal / emergency. |
| Significance | Rating against your criteria. |
| Controls | Operational controls in place. |
| Linked objective | If an objective addresses it. |

## In Trainovate

Maintain this on the **EMS** tab under *Significant environmental aspects*. Each row carries a conformity result so the auditor can record whether controls are adequate and effective.
